# AI Agent 자율성 설계

## 1. Co-pilot이 아니라 Agent인 이유

Co-pilot은 사용자가 매 단계 지시해야 하지만, 본 시스템은 목표를 입력받으면 시나리오 정의, 검증, 실행 계획, 산출물 경로 생성까지 상태를 유지하며 진행합니다. 즉 “도와주는 도구”를 넘어서 “평가 workflow를 스스로 구성하는 Agent”를 지향합니다.

## 2. Agent가 스스로 하는 일

- 자연어 요청에서 시나리오 유형 판단
- 정의서 형식에 맞는 JSON 생성
- 필수값 누락 여부 점검
- 시나리오 1/2에 맞는 OpenCDA 템플릿 선택
- KPI 계산 정책 선택
- 실행 승인 필요 여부 판단
- run_id, manifest, dashboard preview 생성

## 3. Agent가 직접 하지 않는 일

- KPI 수치를 추측해서 만들지 않음
- 원본 OpenCDA 파일을 직접 수정하지 않음
- CARLA 실행을 무승인으로 강행하지 않음
- 평가 기준을 사용자 몰래 변경하지 않음

## 4. 향후 자율화 단계

| 단계 | 내용 |
| --- | --- |
| Level 1 | 계획 생성 및 산출물 폴더 생성 |
| Level 2 | 승인 후 OpenCDA/CARLA 자동 실행 |
| Level 3 | KPI 계산 및 dashboard 자동 생성 |
| Level 4 | 실패 원인 분석 및 재실험 파라미터 제안 |
| Level 5 | 여러 시나리오/속도/통신반경 sweep 후 최적 조건 도출 |

## 5. 평가기관용 설계 포인트

평가기관에는 차량 제어권을 넘기는 것이 아니라, 시뮬레이션과 평가 로그를 자동 정리하는 “평가 자동화 Agent”로 제안하는 것이 적절합니다. 차량 제조사는 시험 대상 알고리즘을 블랙박스 형태로 제공하고, 평가기관은 동일한 시나리오와 동일 KPI로 비교 평가할 수 있습니다.
